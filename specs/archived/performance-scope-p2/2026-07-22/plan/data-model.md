# No Change — Data Model

## Цель

Фича не вводит и не изменяет типы данных, protobuf-схемы, JSON/конфигурационные форматы, структуры БД или межпроцессные контракты.

## Обоснование

Все изменения — замена механизма синхронизации и аллокаций внутри существующих функций/методов:
- `make([]byte, n)` → `sync.Pool.Get` / `sync.Pool.Put`
- `sync.Mutex` → `sync.Map`
- `crypto/rand` → `math/rand/v2`
- Одна RWMutex → два mutex'а

Никакие экспортированные типы, интерфейсы, сигнатуры функций или сериализованные форматы не меняются.

## Соответствие spec

- AC-001–AC-010 все behavioural, не затрагивают data model.
- scope spec явно исключает изменение публичного API.
- Никаких новых сущностей, migration'ов или rollback-шагов не требуется.
